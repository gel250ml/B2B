# B2B для проекта по  "Cовременные решения на базе Python (АО Точка, RUS, ТК)"

**Команда "Гель Фа men 250 мл"**

**Архитектура:**
```
B2BProject/
├── Dockerfile
├── README.md
├── docker-compose.yml
├── migrations/
└── src/
    ├── core/
    │   ├── config.py
    │   └── exceptions.py
    ├── database/
    │   ├── base.py
    │   ├── dependencies.py
    │   ├── init_db.py
    │   └── session.py
    ├── main.py
    ├── models/
    ├── repositories/
    ├── routes/
    ├── schemas/
    └── services/
```

**Подробнее про каждый слой:**
* **core/** - Содержит общие настройки, исключения и утилиты, не зависящие от бизнес-логики и базы данных.

* **database/** - Отвечает за подключение к базе данных, создание сессий и инфраструктурные зависимости для работы с БД.

* **models/** - Содержит ORM-модели, описывающие структуру таблиц базы данных без бизнес-логики.

* **schemas/** - Содержит Pydantic-схемы для валидации и сериализации входящих и исходящих данных API.

* **repositories/** - Реализует доступ к данным и инкапсулирует все SQL-запросы без бизнес-логики и управления транзакциями.

* **services/** - Содержит бизнес-логику приложения, управляет транзакциями и координирует работу репозиториев.

* **routes/** - Определяет HTTP-эндпоинты, обрабатывает запросы и делегирует выполнение бизнес-логики сервисам.

* **main.py** - Точка входа приложения, где создаётся FastAPI-инстанс и подключаются роуты и зависимости.


**Примечание:**
На текущем этапе не используем alembic и migrations.
